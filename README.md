# MetaRefund — TON → USDT Exchange MVP

MetaRefund — интерактивный прототип обменника TON → USDT с пользовательским кабинетом, KYC, тикет-системой и обзором админ-панели. Проект собран как статический фронтенд (HTML/CSS/JS) и иллюстрирует ключевые требования для перехода из MVP в продакшен.

## Возможности прототипа

- **Подключение кошелька**: Tonkeeper, WalletConnect, Trust Wallet и fallback на custodial-аккаунт. Флоу авторизации с подписью challenge и отображением адреса.
- **Панель курсов**: симуляция курса TON → USDT, 24h volume и количества сделок, автообновление каждые 10 секунд.
- **Личный кабинет**: отображение профиля, балансов, истории операций, триггеры Deposit/Swap/Withdraw/History.
- **Swap/Withdraw**: формы с расчётом комиссии, симуляцией прогресса операции и обновлением метрик админ-панели.
- **KYC**: переключение статусов Basic → Verified/Rejected, счётчик событий и связь с журналом.
- **Поддержка**: создание тикетов, история обращений, подсказки по SLA.
- **Админ-панель**: метрики активных пользователей, pending swaps/withdrawals, таблица операций и список интеграций.
- **План внедрения**: модальное окно с поэтапным roadmap от прототипа до продакшена.

## Локальный запуск для проверки

1. Установите [Node.js](https://nodejs.org/) или любой статический http-сервер.
2. В корне репозитория выполните:
   ```bash
   npx serve MetaRefund
   ```
3. Откройте `http://localhost:3000` (или порт из вывода команды) в Google Chrome.

Альтернатива без Node.js — двойной клик по `index.html`. Однако при прямом открытии из файловой системы часть функционала (например, копирование в буфер обмена) может быть ограничена браузером.

## Публикация через GitHub Pages

1. Создайте публичный репозиторий на GitHub и загрузите содержимое папки `MetaRefund`.
2. В настройках репозитория включите **Pages** (Source → `Deploy from a branch`, Branch → `main / (root)`).
3. Через 1–2 минуты получите ссылку `https://<логин>.github.io/<репозиторий>/MetaRefund/` и откройте её в Chrome.

## Альтернативные хостинги

- [Vercel](https://vercel.com) — импортируйте репозиторий, выберите фреймворк «Other», укажите `MetaRefund` как корневую папку.
- [Netlify](https://www.netlify.com/) — «Deploy manually» и загрузка zip-архива с папкой `MetaRefund`.
- Любой S3-совместимый бакет с CDN (CloudFront, R2) для дальнейшего масштабирования.

## Следующие шаги для продакшена

- Определить модель хранения средств (custodial vs non-custodial) и соответствующие юридические требования.
- Интегрировать реальные SDK: Tonkeeper, WalletConnect v2, KYC-провайдер (Sumsub/Onfido), DEX (STON.fi).
- Настроить backend-стек (FastAPI/Node.js), PostgreSQL, Redis, очередь задач и мониторинг (Prometheus + Grafana, Sentry).
- Подготовить CI/CD (GitHub Actions) и инфраструктуру (Docker Compose → Kubernetes), подключить Vault/KMS для ключей.
- Провести аудит смарт-контрактов, пен-тест и подготовить процедуры AML/инцидентов.

Такой план позволит быстро превратить демонстрационный прототип в готовый продукт для обмена TON → USDT.
